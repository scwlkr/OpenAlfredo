# OpenAlfredo Brand Identity System

Brand Guardian: Antigravity  
Strategy Date: 2026-04-05  
Version: 2.0

## Brand Foundation

**Purpose**  
OpenAlfredo exists to make AI interaction feel continuous, dependable, and owned by the user instead of disposable, stateless, and rented.

**Vision**  
A world where every developer can keep a capable local agent that remembers context, maintains momentum, and works in the open.

**Mission**  
OpenAlfredo is a local-first agent workspace that turns one-off prompting into an ongoing conversation backed by memory, initiative, and calm operational discipline.

**Brand Promise**  
You return to context, not a blank slate.

## Brand Values

1. **Continuity**  
Identity, memory, and intent should survive restarts, tabs, and time away.
2. **Openness**  
The system should be inspectable, hackable, and understandable by the people who run it.
3. **Stewardship**  
The agent should feel reliable and organized, not theatrical or reckless.
4. **Sovereignty**  
Your data, your models, your machine.
5. **Measured Initiative**  
The system can wake up and act, but it should do so with restraint and purpose.

## Brand Personality

OpenAlfredo should read and feel like:

- steady
- lucid
- capable
- warm without being cute
- technically candid

OpenAlfredo should not feel like:

- violent
- edgy for its own sake
- chaotic
- corporate
- theatrical

## Positioning

**Primary audience**  
Developer-builders who want a local agent with memory, task continuity, and background initiative.

**Secondary audience**  
AI power users exploring self-hosted, persistent agent workflows.

**Competitive differentiation**  
OpenAlfredo is not another prompt wrapper. It is a local-first agent system with a persistent identity, a durable task layer, and a background heartbeat that can keep work moving between chats.

**Positioning statement**  
For developers who are tired of re-briefing contextless chat tools, OpenAlfredo is the open local-first agent that keeps conversation, memory, and initiative aligned on your own machine.

## Messaging

**Primary tagline**  
Open memory. Ongoing conversation.

**Secondary tagline**  
Local-first context for work that does not reset.

**Value proposition**  
OpenAlfredo gives you a local agent that remembers who it is, what you are doing, and what still needs attention.

**Approved short form**  
`OAX`

**Approved CLI form**  
`oax`

## Voice And Tone

### Voice characteristics

- **Clear**: explain the system plainly
- **Calm**: confident, never melodramatic
- **Technical**: precise where implementation matters
- **Personal**: speak to one operator, not an abstract market

### Tone guidance

- README and docs: practical, inviting, exact
- CLI output: concise, neutral, operational
- onboarding copy: warm, guided, clear
- errors: direct, specific, immediately actionable

### Vocabulary

Prefer:

- agent
- steward
- local-first
- memory
- continuity
- ambition
- heartbeat
- conversation

Avoid:

- kill
- death
- violent iconography
- gore-coded metaphors
- chatbot
- gimmick language

## Visual Identity

### Core palette

| Token | Hex | Use |
|---|---|---|
| `--oax-paper` | `#F4EEDF` | app background |
| `--oax-shell` | `#FFFAF1` | cards and chat surfaces |
| `--oax-ink` | `#171512` | primary text and dark surfaces |
| `--oax-basil` | `#2F6B4F` | primary actions and active states |
| `--oax-basil-strong` | `#254F3C` | hover and pressed states |
| `--oax-brass` | `#B98A3D` | highlights, timing, accents |
| `--oax-sage` | `#8FA08E` | secondary text on dark surfaces |
| `--oax-edge` | `#D7CAB3` | borders and separators |
| `--oax-muted` | `#666257` | supporting copy |

### Visual direction

- warm, open, grounded surfaces
- strong contrast without alarm colors
- crisp geometry with softened edges
- no violent iconography
- no horror metaphors

### Typography

- Primary: Geist Sans
- Monospace: Geist Mono
- Use weight and spacing for hierarchy before decorative treatment

## UI Rules

- Default to parchment and ink, not black and red.
- Use basil for primary actions.
- Use brass for schedule, timing, and emphasis.
- Reserve red for genuine error states only.
- Keep dashboards structured and composed.

## Naming Rules

- Full product name: `OpenAlfredo`
- Technical abbreviation: `OAX`
- CLI and command examples: `oax`
- App package path: `oax-web`

Do not introduce:

- legacy project names
- legacy short-form identifiers
- violent taglines or iconography

## Brand Protection

- Audit new docs and UI strings for stale pre-rename references before release.
- Check package metadata, repository URLs, and CLI docs together when renaming.
- Keep `OpenAlfredo`, `OAX`, and `oax` mapped consistently across user-facing copy, technical identifiers, and command examples.
